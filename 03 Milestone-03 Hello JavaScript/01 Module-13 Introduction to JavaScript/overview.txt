üî¢ VARIABLES AND DATA TYPES:

1. Variable Declaration:
   - Using `var` keyword
   - Variable initialization and reassignment

2. Data Types:
   - Numbers (integers and floats)
   - Strings (text data)
   - Booleans (true/false values)
   - Using `typeof` operator to check data types

3. Type Conversion:
   - `parseInt()` - converts to integer
   - `parseFloat()` - converts to floating-point number

üìù VARIABLE NAMING RULES:
-------------------------
1. Restrictions:
   - Cannot use JavaScript keywords as variable names
   - No spaces or gaps in variable names
   - No quotes in variable names
   - Cannot start with a number (but can contain numbers after first character)

2. Case Sensitivity:
   - Variable names are case-sensitive
   - `address`, `Address`, and `ADDRESS` are different variables

3. Naming Conventions:
   - snake_case: my_current_home_address
   - camelCase: myCurrentHomeAddress (most common in JavaScript)
   - PascalCase: MyCurrentHomeAddress

üßÆ OPERATORS AND SPECIAL VALUES:
--------------------------------
1. Assignment Operators:
   - Basic assignment (=)
   - Compound operators: +=, -=, *=
   - Example: `price += 10` is equivalent to `price = price + 10`

2. String Operations:
   - String concatenation using + operator

3. Special Number Cases:
   - Floating-point precision issues (0.1 + 0.2 = 0.30000000000000004)
   - Using `.toFixed()` method to format decimal places
   - Division by zero results in Infinity

4. Console Operations:
   - Using `console.log()` to display output
   - Debugging variable values in console
